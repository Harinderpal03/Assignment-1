# Reflection

## 1. Can I explain what my code does ?

- This project is a personal resume webpage built with HTML and semantic elements to keep the layout organized and accessible. It includes sections like About, Skills, Education, Projects, and Contact Information, along with navigation links, a profile image, and professional profile links to create a simple and user-friendly online resume.

## 2. What was my coding process ? 

I started by setting up the project folder and creating the basic HTML structure for the webpage. Then, I built the resume section by section, including navigation, contact details, about me, skills, education, experience, and projects. After completing the content, I reviewed the code carefully, fixed any errors, and improved accessibility by adding proper labels and descriptive alt text where needed.


## 3. What challenges did I have ? 

One difficulty I experienced during this project was keeping the webpage layout clean and organized while adding different sections. I also spent time figuring out how to make the page more user-friendly and responsive. Another small issue was making sure images displayed correctly and that all navigation links connected to the right sections. I solved these problems by reviewing my code regularly, testing the webpage in the browser, and making improvements step by step.


## 4. What would I do differently now ?

If I had to start this project from scratch I would plan the webpage structure better before coding. I would create a layout plan to organize sections more efficiently.

Also I would add projects and professional details to make my resume more complete. This would make it more effective. I would set up the HTML structure to make adding CSS styles easier later.

Overall this project helped me understand HTML better. I learned about webpage accessibility and the importance of development practices.
